Tüm admin React CSS breakpoint'leri, mobil cihazlarda WordPress admin kenar çubuğu genişlik ofsetini hesaba katan doğru değer olan **782 piksel**'de standartlaştırıldı.

<!--truncate-->

## Ne Değişti

### Neden 782 Piksel?

WordPress, mobil cihazlarda admin kenar çubuğundan `window.innerWidth`'e ~291 piksel ekliyor. Daha önce kullanılan 768 piksel breakpoint'i çok erken tetikleniyordu — kenar çubuğu hâlâ görünürken, yaygın mobil genişliklerde (375 piksel, 390 piksel, 414 piksel) düzen çakışmaları oluşuyordu.

`782 piksel`, `wp-admin.css` içinde WordPress'in kendi tanımladığı admin breakpoint'iyle eşleşiyor ve tüm admin sayfalarında tutarlı daraltma davranışı sağlıyor.

### Güncellenen Sayfalar

- Kontrol Paneli widget satırları artık ≤ 782 pikselde tek sütuna geçiyor
- Mesajlar tablosu yığılmış düzeni: 768 piksel → 782 piksel
- Raporlar tarih filtresi dikey istifi: 768 piksel → 782 piksel
- Paylaşılan KPI stats kılavuzu: 768 piksel → 782 piksel (2 sütun), 480 piksel (1 sütun, değişmedi)

### Paket

Bu sürüm ayrıca v4.46.0 veya öncesinden güncelleyen operatörler için v4.47.0 (fatal hata düzeltmesi) ve v4.48.0 (mobil CSS düzeltmeleri) içeriyor.
